# Code of Conduct

We want this project to be a welcoming and useful place for people building API tooling.

## Expected behavior

- Be respectful and constructive.
- Assume good intent, especially when discussing bugs or design trade-offs.
- Keep feedback focused on the work, not the person.
- Share enough context for others to reproduce issues.
- Do not post secrets, private keys, tokens, customer data, or production payloads.

## Unacceptable behavior

- Harassment, threats, or personal attacks.
- Discriminatory language or behavior.
- Publishing private information without consent.
- Spamming issues or discussions.
- Intentionally misleading bug reports or exploit instructions.

## Reporting

If you see behavior that violates this code, report it privately to the project maintainers. If no private contact is configured yet, open a GitHub issue asking maintainers to add a private reporting channel without including sensitive details.

Maintainers may remove comments, close issues, block users, or take other action needed to keep the project healthy.
